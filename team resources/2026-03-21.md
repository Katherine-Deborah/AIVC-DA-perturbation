

## Better Evaluation Design For Perturbation Prediction

### Why The Old Evaluation Is Not Good Enough

The first custom `STATE` evaluation we added reports:

- `Pearson r (all genes)`
- `Pearson r (top-20 DEGs)`
- `Pearson r (DA markers)`

This is useful as a first check, but it is not yet strong enough for the real scientific goal.

Main problem:

- very high Pearson on `all genes` can happen even when the model is mostly predicting the
  average cell-type background correctly rather than the perturbation effect itself

Why:

- most genes change only a little under a single perturbation
- cell identity explains much more variance than the perturbation does
- group-level mean expression vectors are already very similar to each other

So a result like:

- high `Pearson r (all genes)`
- high `Pearson r (DA markers)`
- much lower `Pearson r (top-20 DEGs)`

should be interpreted carefully.

It may mean:

- the model captures the overall neuronal background well
- but is still much weaker on perturbation-specific transcriptional change

The current `DA markers` metric has a similar limitation:

- many DA markers may be relatively stable identity markers rather than strong perturbation-response genes
- a high Pearson on the DA-marker panel can therefore reflect good global state matching more than good causal perturbation prediction

### What The Better Evaluation Should Measure

A stronger evaluation should separate:

1. overall expression similarity
2. perturbation-specific change from control
3. correctness on biologically relevant DA-marker responses

The key idea is:

- do not only compare `predicted perturbed` vs `true perturbed`
- also compare the predicted *delta from control* vs the true *delta from control*

This follows the same logic described in the project report for GEARS, where:

- `Pearson delta` is considered the more stringent perturbation-specific metric

### Recommended New Metrics

For each held-out perturbation and cell type:

1. compute the control mean expression vector
2. compute the true perturbed mean expression vector
3. compute the predicted perturbed mean expression vector
4. define:
   - `true_delta = true_pert_mean - control_mean`
   - `pred_delta = pred_pert_mean - control_mean`

Then report these metrics:

#### A. Keep the old absolute metrics

- `Pearson r (all genes)`
- `Pearson r (top-20 DEGs)`
- `Pearson r (DA markers)`

These remain useful, but should not be the main headline metrics.

#### B. Add delta-based metrics

- `Pearson delta (all genes)`
- `Pearson delta (top-20 DEGs)`
- `Pearson delta (DA markers)`

These are better because they ask:

- did the model predict the *change caused by perturbation* rather than just the background cell state?

#### C. Add direction-of-change accuracy

For the same gene subsets, compute:

- fraction of genes whose predicted change direction matches the true change direction

Suggested versions:

- `sign accuracy (top-20 DEGs)`
- `sign accuracy (DA markers)`

This matters because a model can have decent correlation while still predicting the wrong
up/down direction for important genes.

#### D. Add effect-size error

For the same gene subsets, compute:

- `MSE delta (top-20 DEGs)`
- `MSE delta (DA markers)`

This helps distinguish:

- right direction but wrong magnitude
- correct effect size

### Recommended Final Evaluation Package

For each model, the most informative report table should emphasize:

- `Pearson delta (top-20 DEGs)`
- `Pearson delta (DA markers)`
- `sign accuracy (top-20 DEGs)`
- `sign accuracy (DA markers)`

And keep these as secondary context metrics:

- `Pearson r (all genes)`
- `Pearson r (DA markers)`
- `MSE`

### Why This New Evaluation Is Better

The new design is better because it separates two different questions:

- does the model reconstruct the overall cell state?
- does the model reconstruct the perturbation effect?

The first question is easier and often gives inflated-looking scores.

The second question is the one that matters for:

- zero-shot perturbation prediction
- perturbation ranking
- identifying interventions that move cells toward or away from a DA-neuron-like program

### Practical Recommendation

Going forward:

- keep the current absolute Pearson metrics
- do not use them alone as the main claim
- add delta-based evaluation as the main perturbation-specific benchmark
- treat DA-marker delta metrics as more biologically meaningful than DA-marker absolute Pearson

### Next Implementation Step

The next evaluation upgrade should add:

- `pearson_delta_all_genes`
- `pearson_delta_top20_degs`
- `pearson_delta_da_markers`
- `sign_accuracy_top20_degs`
- `sign_accuracy_da_markers`
- `mse_delta_top20_degs`
- `mse_delta_da_markers`

to the custom `STATE` evaluation script so the benchmark aligns better with the project's actual
scientific objective.

## Two-Track Experiment Design

The current project has two related but distinct goals:

- build a strong practical perturbation model that can eventually reason about combinations
- evaluate whether the model can generalize to unseen cell types / stages without data leakage

These goals should be handled with two separate model tracks.

### Track A: Clean Zero-Shot Evaluation

Use:

- base model: `ST-SE-Replogle`
- fine-tuning data: `GSE152988` with both `CRISPRi` and `CRISPRa`
- test dataset: `GSE124703`

Purpose:

- test `zero-shot stage`
- test cell-type / developmental-context transfer using the current `GSE124703` split design

Important rule:

- `GSE124703` must not be used in training before this benchmark is completed

Reason:

- if the model has already seen `GSE124703`, then evaluation on `GSE124703` is no longer a clean
  zero-shot test and can be affected by dataset leakage

Interpretation:

- if the `ST-SE-Replogle -> GSE152988 (CRISPRi + CRISPRa)` model performs well on `GSE124703`
  `zero_shot_stage` and related unseen-context tests, that is evidence the model can
  generalize across unseen cellular contexts

### Track B: Continued Fine-Tuning for the Final Application Model

After Track A evaluation is finished, it is reasonable to continue training with `GSE124703`.

Use:

- initialize from the Track A model
- continue fine-tuning on `GSE124703`
- evaluate on a new dataset or new held-out cell type that has not been used yet

Purpose:

- build a stronger final model for perturbation ranking and combination design
- expose the model to both repression-like and activation-like perturbation regimes
- support future reprogramming-style applications where up-regulation matters, not only knockdown

Important rule:

- once `GSE124703` is used for continued fine-tuning, it can no longer serve as the clean
  zero-shot benchmark dataset

### Imbalance Risk When Adding GSE124703

`GSE124703` currently contributes only `CRISPRi`-style perturbations and does not provide a
matching `CRISPRa` branch.

This does not create label confusion in the current preprocessing design, because:

- `CRISPRI::GENE` and `CRISPRA::GENE` are treated as different perturbations
- controls remain unified as `control`

However, it can introduce a training-distribution bias:

- the model may see many more repression-like perturbations than activation-like perturbations
- the model may become better calibrated for knockdown / repression than for activation /
  overexpression-like interventions
- this matters for the long-term goal of reprogramming-style perturbation design, where gene
  activation is biologically important

So the main risk is not semantic corruption of the perturbation labels. The main risk is
imbalance between perturbation directions.

### Practical Bias-Mitigation Ideas

If `GSE124703` is added later for continued fine-tuning, the following mitigation steps are worth
considering:

- keep `GSE152988 CRISPRa` data in the training mixture so activation examples are not lost
- monitor metrics separately for `CRISPRi` and `CRISPRa` style perturbations instead of relying
  only on one overall score
- consider reweighting or oversampling the activation branch if `CRISPRa` examples become too rare
- keep reporting evaluation results by perturbation direction whenever possible
- for future combination-design experiments, explicitly check whether activation-like predictions
  degrade after adding knockdown-heavy datasets

### Practical Conclusion

Recommended sequence:

1. fine-tune `ST-SE-Replogle` on `GSE152988` using both `CRISPRi` and `CRISPRa`
2. evaluate that model on `GSE124703` for `zero-shot stage` and related unseen-context transfer
3. freeze the benchmark conclusion from that evaluation
4. only after that, continue fine-tuning with `GSE124703` if desired
5. use a different unseen dataset or unseen cellular context for the next generalization test

This separation keeps the benchmark scientifically clean while still allowing a stronger final
model for downstream DA-neuron perturbation design.
